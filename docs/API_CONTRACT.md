# API_CONTRACT.md

## Argus API Contract Source of Truth (Alpha MVP)

**Status:** Active | **Alpha Contract v1 Locked**
**Audience:** Backend engineers, frontend engineers, AI agents
**Purpose:** Define the current Alpha MVP interfaces between client and server. All frontend/backend integration should follow this document. Code should bend to the contract unless intentionally revised.

> [!IMPORTANT]
> **Locked Status**: No breaking changes (structural JSON shape changes) are allowed without explicit approval or critical implementation blockers. Additive changes and clarifications are permitted.

---

# 1. Contract Principles

## Current Scope Only

This document reflects the Alpha MVP product, not future vision.

## Explicit > Implied

No undocumented payload shapes.

## Stable Naming

Use consistent field names across endpoints.

## Forward Evolvable

Prefer shapes that can grow without breaking clients.

## Secure by Default

Authenticated endpoints require valid session.

## AI-Native Defaults

Names and titles should be AI-generated by default where context exists, while allowing users to rename later.

## Metrics Context vs. Engine Canonicality

Metrics are context-specific at the presentation layer but canonical at the engine layer. Engine result metrics are canonical; chat result cards are beginner-friendly aggregate presentations; strategy surface metrics are user-configurable symbol-level presentations.

## Reproducibility

Every backtest run must be reproducible from its `config_snapshot`. Defaults must be persisted, and result cards must show the assumptions used during execution.

## Conversational Onboarding

Onboarding is a conversational state machine, not a rigid form wizard. AI can guide progression dynamically, and the profile state exists to personalize prompts and improve recommendations.

## Ask Only What Improves the Next Useful Response

Do not block users with friction. Language improves comprehension; primary goal improves the quality of the first AI prompt. Everything else should be delayed until contextually useful or derived through conversation.

## Language & Locale Consistency

Language preference is global, persistent, and reversible. Changing language updates the UI and AI response language, while the locale controls formatting for dates, numbers, and currency.

# 3. General Conventions

## Pagination

All list endpoints must use cursor-based pagination.

**Cursor format:**
`opaque base64(timestamp + id)`

**Required for:**
- `GET /conversations`
- `GET /conversations/{id}/messages`
- `GET /strategies`
- `GET /collections`
- `GET /history`
- `GET /search`
- future archived/deleted lists

**Standard response shape:**
```json
{
  "items": [],
  "next_cursor": null
}
```

## Rate Limiting

Every API response should include rate-limit headers where applicable:
- `X-RateLimit-Limit`
- `X-RateLimit-Remaining`
- `X-RateLimit-Reset`
- `Retry-After`

## Asset Classes (Alpha Enum)
- `equity`
- `crypto`

## Backtest Status (Alpha Enum)
- `queued`
- `running`
- `completed`
- `failed`

## Idempotency
Alpha supports server-side idempotency for expensive state-changing operations.
- **Header**: `Idempotency-Key`
- **Duration**: 24 hours.
- **Scope**: `authenticated user_id` + `endpoint` + `Idempotency-Key`.
- **Behavior**: Identical requests (same key/user/endpoint) return the same response or current known status. Collisions across different users are prevented by user-scoping.

## Admin Bypass

If `profiles.is_admin = true`, backend quota and rate-limit restrictions are bypassed.

> [!IMPORTANT]
> Admin bypass must never bypass ownership/RLS privacy rules.

## Error Responses

Errors follow RFC 9457 Problem Details.

**Standard shape:**
```json
{
  "type": "https://api.argus.app/problems/validation-error",
  "title": "Validation Error",
  "status": 400,
  "detail": "Symbol is required.",
  "code": "validation_error",
  "request_id": "uuid"
}
```

*Note: Success responses should also include `X-Request-Id` in the response headers.*

**Error Status Rules:**
- **400 Bad Request**: Invalid input
- **401 Unauthorized**: Missing or invalid auth
- **403 Forbidden**: Disabled feature or insufficient access
- **404 Not Found**: Missing resource or inaccessible resource
- **409 Conflict**: State conflict
- **422 Unprocessable**: Valid JSON but invalid domain request
- **429 Too Many Requests**: Rate limit exceeded
- **500 Server Error**: Unexpected failure

**Example 422 Problem Details:**
```json
{
  "type": "https://api.argus.app/problems/unsupported-timeframe",
  "title": "Unsupported Timeframe",
  "status": 422,
  "detail": "Supported timeframes are 1h, 2h, 4h, 6h, 12h, and 1D. Lookback cannot exceed 3 years.",
  "code": "unsupported_timeframe",
  "request_id": "uuid"
}
```

## Display Date Conventions

Machine-readable values must remain ISO 8601 (YYYY-MM-DD). User-facing cards and explanations must use readable display labels that respect the user's `locale`.

**Format:** `MonthName DD, YYYY` (or locale equivalent)
**Examples:**
- `en-US`: "January 1, 2022 to December 31, 2024"
- `es-419`: "1 de enero de 2022 al 31 de diciembre de 2024"

```json
{
  "date_range": {
    "start": "2025-04-23",
    "end": "2026-04-23",
    "display": "April 23, 2025 to April 23, 2026"
  }
}
```

## Static UI vs. AI Response Translation

The API stores and returns language/locale preferences. Static UI translation is handled by the frontend i18n layer. AI response language is handled by backend prompt/orchestration using the resolved user language.

**Reserved for later monetization:**
- **402 Payment Required**: Quota exhausted or paid entitlement required

> [!TIP]
> **Global Rule**: Collections may mix asset classes organizationally. Backtest runs may not mix asset classes operationally.

# 6. Backtest Input Contract

The canonical backtest config used by the engine for execution and reproducibility.

```json
{
  "template": "rsi_mean_reversion",
  "asset_class": "equity",
  "symbols": ["AAPL"],
  "timeframe": "1D",
  "start_date": "2025-04-23",
  "end_date": "2026-04-23",
  "side": "long",
  "starting_capital": 10000,
  "allocation_method": "equal_weight",
  "benchmark_symbol": "SPY",
  "parameters": {}
}
```

## Required vs. Defaulted Fields

**Required by runtime after normalization:**
- `template`, `asset_class`, `symbols`, `timeframe`, `start_date`, `end_date`, `side`, `starting_capital`, `allocation_method`, `benchmark_symbol`, `parameters`.

**Backend Defaults (if omitted):**
- `timeframe`: "1D"
- `start_date`: Rolling 12 months before `end_date`
- `end_date`: Latest available market date
- `side`: "long"
- `starting_capital`: 10000
- `allocation_method`: "equal_weight"
- `benchmark_symbol`: `SPY` (equity), `BTC` (crypto)
- `parameters`: Template-specific defaults

## Constraints & Validation (Alpha MVP)

### Side / Direction
- **Allowed:** "long"
- **Rejected:** "short", "long_short", "market_neutral"
- *Return 422 for unsupported values.*

### Starting Capital
- **Default:** 10000
- **Allowed Range:** 1,000 to 100,000,000
- *Return 422 for values outside range.*
- > [!NOTE]
  > Starting capital is simulation capital only. It does not imply real brokerage trading or account balance.

### Symbol Constraints
- **Minimum:** 1 symbol
- **Maximum:** 5 symbols
- **Rules:** Same asset class required. Cross-asset multi-symbol backtests are deferred.
- *Return 422 for violations.*

### Timeframe & Lookback
- **Supported:** "1h", "2h", "4h", "6h", "12h", "1D"
- **Constraints:** No timeframe lower than "1h". Lookback cannot exceed 3 years. Daily defaults to a 12-month window when dates are omitted.
- *Return 422 for unsupported combinations.*

### Date Range
- **Format:** ISO 8601 (YYYY-MM-DD)
- **Rules:** `start_date` must be before `end_date`. `end_date` cannot be in the future beyond latest market data.
- *Normalization:* Backend computes rolling 12 months if dates are omitted.

## Reality Gap (Deferred)
Argus Alpha is a "perfect world" simulation. The following are explicitly excluded from Alpha MVP:
- slippage, fees, order queue modeling, liquidity/spread modeling, volatility hazards.

# 9. Language & Locale Resolution

Argus supports English and Spanish (Latin America) in Alpha.

## Supported Values
- **language:** `en`, `es-419`
- **locale:** `en-US`, `es-419`

*Note: `language` controls the AI response language and the static UI preference. `locale` controls formatting (dates, numbers, currency).*

## Source of Truth Rules
- **Authenticated Users**: `profiles.language` and `profiles.locale` are the persisted source of truth. Profile preference wins over browser detection.
- **Logged-out Users**: Frontend may store preferences in `localStorage`. API does not persist these unless guest sessions are implemented.

## Frontend Resolution Order
1. Authenticated user profile preference
2. Saved local preference (`localStorage`)
3. Browser language (`navigator.language`)
4. Fallback to English (`en` / `en-US`)

# 10. Onboarding State Machine

Alpha onboarding is designed to reduce friction and get users into the chat experience immediately. The real onboarding is the first successful backtest.

## Supported Onboarding Stages
- `language_selection`
- `primary_goal_selection`
- `ready`
- `completed`

*Note: User should reach the chat surface immediately after language and primary goal selection.*

## Primary Goals (Alpha Enum)
The `primary_goal` field routes the initial AI behavior and reduces "blank page" friction.

- `learn_basics`: Suggest beginner-safe concepts and simple explainers.
- `test_stock_idea`: Invite user to name a symbol or investment thesis.
- `build_passive_strategy`: Suggest long-term DCA or index-tracking starters.
- `explore_crypto`: Prioritize crypto-supported templates and symbols.
- `surprise_me`: Provide guided, diverse starter prompts.

## AI Behavior Guidance
After onboarding, the AI should use the `primary_goal` to personalize the first prompt:

- **learn_basics** -> "Welcome! Let's start simple. Want to see how a basic 'Buy the Dip' strategy works with Apple?"
- **test_stock_idea** -> "I'm ready. What's a stock or crypto you're watching, and what's your idea for it?"
- **explore_crypto** -> "Crypto markets are active. Want to test an RSI strategy on Bitcoin or Solana?"

## Product & Architecture Notes
- **Frictionless**: Do not collect age, phone numbers, or complex risk tolerance profiles in Alpha.
- **Conversational**: Further personalization happens contextually inside the chat.
- **Storage**: Onboarding state belongs in the canonical Supabase profile store.
- **Future Compatibility**: Richer preferences (watchlists, memory) may be added later as conversation-derived data.

# 12. Chat Streaming Endpoint

**Request:**
```json
{
  "conversation_id": "uuid",
  "message": "Quiero probar comprar cuando baja Tesla",
  "language": "es"
}
```

**Required Header:**
- `Idempotency-Key`: `uuid` (Highly recommended to prevent double-submits)

**Language Resolution:**
Backend resolves preferred response language from:
1. Explicit request `language`
2. Authenticated user profile `language`
3. Fallback English

AI response uses the resolved language unless the user clearly asks otherwise in-message.

**Response Transport:**
`Content-Type: text/event-stream`

**Reconnect Behavior:**
If the stream disconnects, the client should reconnect by re-fetching the conversation messages (`GET /conversations/{id}/messages`).
- No token-level replay is guaranteed in Alpha.
- No `last_message_id` reconnect contract is enforced yet.

**Recommended Header:**
- `X-Accel-Buffering: no` (To prevent proxy buffering)

**Stream Events:**

- **event: token**
  - `data`: partial ai text
- **event: title**
  - `data`: `{"conversation_id":"uuid","title":"Tesla Dip Strategy"}`
- **event: status**
  - `data`: `{"status":"extracting_strategy"}`
- **event: status**
  - `data`: `{"status":"running_backtest"}`
- **event: result**
  - `data`: `{"run": {...}}`
- **event: done**
  - `data`: `{"message_id":"uuid"}`

*Note: Server may include `retry: 3000` in the stream, but clients should implement their own backoff/reconnect logic.*

# 17. Search

## `GET /search`

Global omni-search.

**Query Params:**
- `q`
- `limit`
- `cursor`

**Response:**
```json
{
  "items": [
    {
      "type": "chat",
      "id": "uuid",
      "title": "Tesla Dip Strategy",
      "matched_text": "...",
      "updated_at": "timestamp"
    }
  ],
  "next_cursor": null
}
```

**Ranking Logic:**
Results are ranked by:
1. **Pinned Boost**: Pinned items always appear first.
2. **Exact Match**: Full title/name match.
3. **Symbol Match**: Match against symbols in strategies/backtests.
4. **Recency**: Sorted by `updated_at` within same relevance tier.
5. **Basic Text Relevance**: Keyword matching in metadata.

**Search Scope:**
Search is limited to:
- Titles and Names
- Symbols
- Last message preview (Conversations)
- Collection names
- Strategy template labels (e.g. "Momentum Breakout")

*Note: Alpha search does not perform deep indexing of full message bodies or complex template parameters.*

*Future semantic retrieval may extend this endpoint.*

# 19. Feature Flags

Feature flags may be returned in session/profile responses.

```json
{
  "flags": {
    "notifications": false,
    "subscriptions": false
  }
}
```

*Feature-disabled endpoints should return 403.*

For Alpha, the Settings "Upgrade" button may be shown behind a feature flag as a visual placeholder only. No billing, entitlement mutation, or upgrade API behavior is implemented.

Backend runtime flags may also control internal engine behavior. For Alpha, `ARGUS_ENABLE_EXECUTION_REALISM` exists for staged development and is `false` by default. While disabled, Alpha API behavior and snapshots remain canonical "no fees/slippage" with no public request/response contract expansion.
